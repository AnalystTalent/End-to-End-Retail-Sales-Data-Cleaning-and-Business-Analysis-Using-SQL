End-to-End-Retail-Sales-Data-Cleaning-and-Business-Analysis-Using-SQL
Retail Sales Data Cleaning and Business Analysis using SQL Server. This project involves data cleaning, duplicate removal, handling missing values, standardizing records, and analyzing sales performance, profitability, customer behavior, regional trends, and operational efficiency to generate business insights and support decision-making.

The project covers:
- Data exploration
- Data cleaning
- Duplicate removal
- Missing value handling
- Data standardization
- Business analysis
- Performance insights generation
  
Tools Used
- SQL Server
- SSMS (SQL Server Management Studio)
  
Business Questions Solved
Data Cleaning

- How many total records exist?
- Are there duplicate orders?
- Are there missing customer details?
- Are there invalid sales or profit values?
- How can raw data be standardized for analysis?
- 
Business Analysis

- What is the total sales and total profit?
- Which categories generate the highest profit?
- Who are the top customers by sales?
- Which products generate the highest losses?
- Which region is most profitable?
- Which customer segment drives the most profit?
- What is the average delivery time?
- Which products are slow-moving?
- Which products are heavily discounted?
- Which cities generate high sales but low profit?
- How does yearly sales performance compare?

Key Business Insights

1. Duplicate records were identified and removed using ROW_NUMBER().
2. The West region was the most profitable.
3. Some products generated significant losses.
4. Heavy discounts reduced profitability.
5. Some cities had high sales but low profit.
6. Average delivery time helped evaluate shipping efficiency.

Business Recommendations

1. Review pricing strategy for highly discounted products.
2. Investigate and manage loss-making products.
3. Replicate successful strategies from high-performing regions.
4. Improve margins in high-sales but low-profit cities.
5. Strengthen retention strategies for top customers.
